Getting Started with SweetSpot

Before running your first price test, we need to make sure SweetSpot is set-up correctly.

## Tagging prices
Any element with a price inside should have an identifying class and attribute. In the default theme this means:

1. SweetSpot needs to identify which variant a price belongs to. On pages that show variants we find it on the `variant`, on other pages we use the `product`. Add the following snippet to `product-price.liquid` to help SweetSpot do this.
```liquid
{% if variant.sku %} 
{%- assign sweetspot_id = variant.id -%}
{% else %} 
{%- assign sweetspot_id = product.first_available_variant.id -%}
{% endif %}
```

2. Add the class  `sweetspot__price` to the element containing the price.
3. Add the id as a data attribute: `data-sweetspot-id={{ sweetspot_id }}`.
